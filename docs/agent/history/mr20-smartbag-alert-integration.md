# MR20 SmartBag 四级预警接入

## 目标

将 MR20 网口雷达作为右后风险来源接入现有 SmartBag 控制器，并与摄像头风险按侧融合输出 1-4 级震动和语音提醒。

## 最终方案

- `mr20_radar.py` 解析官方 14 字节 UDP 帧：`0x60A` 聚合一帧目标列表，`0x60B` 解码纵向/横向距离和速度。
- `mr20_radar.example.json` 提供右后默认配置：接收 `192.168.1.102:2368`，仅接受默认雷达 `192.168.1.200`；左后第二颗雷达预留为禁用实例。
- 风险按纵向距离、逼近速度和 TTC 分为 1-4 级；同一目标连续两帧才输出。控制器按 `source + side` 保留事件，侧输出取有效来源的最高等级。

## 验证结果

- 本地 Python 单元测试通过：官方样例、非法帧、`0x60A`、来源过滤、连续帧、TTC、横向过滤、四级等级和摄像头/雷达融合。
- 板端只读探测确认：`eth0=192.168.1.168/24` 正常 SSH；`eth1` 当前为 DOWN 且没有地址，因此尚未配置雷达调试 IP，也未接收到实物 UDP 数据。

## 未验证项与注意事项

- 雷达接入 `eth1` 并显示链路 UP 后，临时添加 `192.168.1.102/24`，不设置网关、不修改 `eth0` 或默认路由。
- 必须依次验证 UDP 抓包、JSONL 日志、`--dry-run`、PWM/语音实物输出；阈值只在实测后调整 JSON 配置。
- 当前资料和 ROS SDK 的纵向轴为解码字段 `datay`，逼近目标的纵向速度为负；若实物安装方向相反，仅调整 `approaching_velocity_sign`，不要交换协议字段。
