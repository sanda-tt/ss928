# SS928 Factory YOLOv8 VOT Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver and verify an AArch64 SS928 program that uses the unchanged factory YOLOv8 OM for offline detection, VOT tracking/distance/relative-speed/risk, JSONL integration, and finally single-camera UVC input.

**Architecture:** A descriptor-driven ACL owner reuses model and tensor buffers across frames. Descriptor-selected preprocessing and validated YOLOv8 decoding feed the copied tracker/VOT core, while input sources remain replaceable and all machine-readable events are isolated on stdout.

**Tech Stack:** C99, C++14/17 as supported by `aarch64-mix210-linux-g++`, SS928 V2.0.2.2 ACL, Linux V4L2, Make, shell, existing VOT C/C++.

## Global Constraints

- Source root: `work/ss928_yolov8_vot/`; board root: `/root/smartbag_alert/runtime/yolov8_vot/`.
- Model: read-only `/opt/sample/yolov8/yolov8n.om`; never retrain, reconvert, replace runtime, or modify the OM.
- Compiler: `/opt/linux/x86-arm/aarch64-mix210-linux/bin/aarch64-mix210-linux-` in WSL2.
- SDK: `在线仓库/SS928V100_SDK_V2.0.2.2_MPP_Sample-master` only.
- No NCNN, ONNX Runtime, Ultralytics, OpenMP, `-fopenmp`, or `libgomp`.
- Tests and fresh logs gate every phase transition.
